  - ビジネスロジック
    - 流れや制御の部分
    - 「予約を受け付けて → 確認メールを送信 → ステータスを更新」
  - ドメインロジック
    - ルールや概念の定義部分
    - 「そもそも予約とは何か」「予約可能条件は何か」
- **ハイドレーションエラーとは**
  - まず、「ハイドレーション（hydration）」とは？
    - React（やNext.js）はSSRでページを生成する際に、サーバー側で HTML を生成してブラウザに送る→ブラウザ側でReactが起動し、JSを読み込む
    - そのHTMLにReactの「機能（イベント・状態）」を紐づける←ハイドレーション
  - React が「サーバーで作ったHTML」と「クライアント（ブラウザ）でレンダリングした結果」が違うときに発生
    - レンダリング結果が動的に変わってしまう（new Date()がサーバー側とクライアント側で差があるなど）
    - ランダム値・一意IDなどが変わる（Math.random()を使用し、サーバー側とクライアント側で乱数が違う）

####  :seedling: 業務メモ
- ReactとReactRouterの違い
  - ReactはUIを作る
  - ReactRouterはページ遷移（ルーティング）を実現する
- Remixとは
  - Reactのフレームワークの一つ
  - ReactRouterの生みの親がReactを離れて作った独自のフレームワーク
  - ルーティング・データ取得・フォーム送信・SSRをまるっとサポート
  - Next.jsと比べて
    - フォーム送信
      - HTMLフォームを書くだけで、サーバー側の action() 関数を呼び出してくれる（Reactだとfetch() や axios.post()が必要）
    - データ取得
      - 「getServerSideProps」や「useEffect + fetch」を使用せず、各ページに loader を書くだけ
- memcachedとは
  - Webアプリケーションの処理を高速化するためのキャッシュサーバー
  - よく使うデータをメモリ上に一時的保存しておく仕組み（アプリ内部）
- E2Eテスト（End-to-End）とは
  - ユーザー視点でのテスト（実際に操作するように自動で）
    - ボタンを押す→API通信が起こる→データが画面に反映される等
  - ユニットテスト
    - 1つの関数・コンポーネント単位
  - 統合テスト
    - 複数モジュールの連携
- Playwright MCPとは
  - ブラウザを操作できる
  - Playwright
    - ブラウザ自動操作・E2Eテスト用フレームワーク
  - MCP
    - AIがいろんなサービスと安全に会話・連携するための共通言語
