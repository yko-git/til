# 2026/4/1
#### :seedling: 技術インプット
- **セキュリティ**
  - [サプライチェーン攻撃から身を守るために最低限設定しておきたいこと](https://zenn.dev/dely_jp/articles/supply-chain-kowai)
  - 対策
  - クールダウンの導入
    - 公開から一定時間が経過するまでインストールを保留する仕組み
    - CI/CD・npm installでも有効
  - ロックファイルの厳密な運用
    - CI/CDでは npm ci を使用する
      - npm ci は不整合があればエラーで止まり、ロックファイルに記録されたバージョンだけをインストールする
  - GitHub Actions のSHA pinning
    - actions/checkout@v4のようなタグ指定は上書きされると別のコードが実行される可能性がある
      - ハッシュで固定（SHA pinning）すると防げる
      - pinactで自動化できる
  - Dependency Review Action の導入
    - PRで新たに追加・変更される依存関係をスキャンし、脆弱なパッケージが含まれていたらCIを失敗させる仕組み
    - マージ前にブロックできる
- **AIドキュメンテーションの運用について**
  - 差分検知検証
  - Claude Code ツール
    - Bash
      - シェルコマンドの実行をする唯一のツール
        - npm run,git,mkdir,ls,cat
    - ファイル操作ツール
      - Claudeが提供する最適化されたツール
      - Read
        - ファイルの内容を読み込む
      - Write
        - ファイルを新規作成・上書き
      - Edit
        - 既存ファイルの一部を置換・差分のみ送信
      - Glob
        - ファイルパターンマッチング
      - Grep
        - ファイルの内容を検索
    - 専用ツールの利点
      - より高速・セキュリティチェック付き・エラーハンドリングが優れている
  - -- bareモード
    - Claude Code CLIのミニマルモード
      - CI/CD 環境での実行に最適化されたモード
    - -- bareで無効化されるもの
      - フック/プラグイン同期/Attribution/Auto-memory/CLAUDE.mdの自動検出
    - 有効なもの
      - SKILLS/Bedrockなど認証/明示的に指定したコンテキスト
    - なぜCIで使うのか
      - 不要な処理をスキップして高速
      - セキュア（キーチェーンやOAuth不使用）
      - シンプル
